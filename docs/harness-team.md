# 하네스 팀 운영 문서

## 목적

이 팀은 "초등학생도 바로 시작할 수 있는 구구단 타임어택 게임"을 만드는 것을 목표로 움직입니다.

게임의 핵심은 아래와 같습니다.

- 1분 동안 최대한 많은 문제를 푼다
- 문제는 랜덤 구구단으로 나온다
- 답은 숫자패드로 입력한다
- 오답은 바로 처리하고 다음 문제로 넘어간다
- 결과 화면에서 기록을 쉽게 보여준다

## 팀 구성

### 1. planner

역할

- 초등학생이 바로 이해할 수 있는 규칙을 정리
- 시작, 플레이, 결과 화면 흐름을 단순하게 설계
- 숫자패드 중심 조작 방식을 명확히 정의

산출물

- 게임 규칙 초안
- 화면 흐름 초안
- 입력 방식 초안

체크 기준

- 설명을 읽고 1분 안에 시작할 수 있는가
- 규칙이 짧고 분명한가
- 점수와 결과가 바로 이해되는가

### 2. developer

역할

- 기획 문서를 바탕으로 UI와 게임 로직 구현
- 문제 생성, 타이머, 입력 처리, 점수 집계를 구현
- GitHub Pages로 바로 배포 가능한 정적 구조로 정리

산출물

- 실행 가능한 게임 화면
- 숫자패드 입력 로직
- 랜덤 문제 및 결과 집계 로직

체크 기준

- 숫자패드가 모바일에서 누르기 쉬운가
- 정답과 오답이 정확하게 집계되는가
- 시간이 끝났을 때 자연스럽게 결과 화면으로 넘어가는가

### 3. tester

역할

- 초등학생 기준으로 문구와 흐름을 점검
- 모바일과 데스크톱 조작 차이를 검수
- 버그와 사용성 문제를 구분해서 전달

산출물

- 시나리오별 점검 기록
- 버그 목록
- 개선 제안 목록

체크 기준

- 처음 보는 아이도 바로 답을 입력할 수 있는가
- 오답 처리 방식이 헷갈리지 않는가
- 결과 화면에서 자신의 기록을 쉽게 읽을 수 있는가

## 협업 흐름

1. `planner`가 규칙과 화면 흐름을 짧게 정리합니다.
2. `developer`가 바로 실행 가능한 정적 게임으로 구현합니다.
3. `tester`가 실제 플레이 기준으로 혼란 요소와 버그를 점검합니다.
4. 수정 후 같은 흐름으로 다시 확인합니다.

## 현재 공통 원칙

- 문장은 짧고 쉽게 쓴다.
- 설정 없이 바로 시작할 수 있게 만든다.
- 빠른 템포와 반복 플레이 재미를 살린다.
- 화면마다 지금 상태를 쉽게 이해할 수 있게 만든다.
